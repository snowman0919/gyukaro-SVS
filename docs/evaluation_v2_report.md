## Frozen neural decoder recovery probe (not yet the packaged hybrid runtime)

`scripts/probe_soulx_score.py` runs one **whole 9.94-second phrase** through
Apache-2.0 ACE-Step lyric-vocal content generation followed by Apache-2.0
SoulX-Singer SVC. SoulX receives the complete, explicit 50 Hz score F0
contour; it does not use per-note TTS, pitch shifting, time stretching, or
waveform concatenation. `scripts/evaluate_soulx_score_probe.py` evaluates
the generated phrases with SoulX RMVPE and Whisper. The deterministic
C4–G4–C5–G4 score test passed all predeclared gates:

| Language | F0 corr | pitch MAE | held-note F0 CV | ASR lyric similarity |
|---|---:|---:|---:|---:|
| KO | 0.9838 | 17.85 cents | 0.0057 | 1.0000 |
| EN | 0.9575 | 41.83 cents | 0.0057 | 0.8158 |
| JA | 0.9793 | 18.14 cents | 0.0069 | 0.7333 |

The reproducible JSON is `artifacts/reports/soulx_score_probe.json`. This
proves a viable quality decoder and multilingual phrase-content route, but it
does **not** repair the compact `hybrid-svs` checkpoint above. It therefore
does not change the v0.2/package status or constitute a v1 claim. Production
runtime integration and a fresh package audit remain required.

## Reconstructed-score retrain: failed quality gate

`checkpoints/gyu_hybrid_v0.3_score_reconstructed.pt` was trained for 8,000
steps after replacing synthetic base-note labels with RMVPE/script-prior score
reconstruction, separating actual F0 targets from score-only pitch inputs, and
adding MMS CTC forced phoneme alignment. Its independent report is
`artifacts/reports/hybrid_score_reconstructed_evaluation.json`: KO/EN/JA F0
correlation was `0.0535/-0.1049/0.0563` and pitch MAE was
`1386.04/1461.64/1427.69` cents. The 3-second hold report also misses C4.
This retrain is a failed experiment, not the production candidate.

## Residual-flow compact retrain: failed quality gate

The v0.4 compact sampler starts from the `SingingDecoder`'s
condition-derived codec-latent source and uses conditional flow only for the
source-to-target residual.  It lowers validation acoustic loss, but generated
KO/EN/JA F0 correlation is `-0.3001/-0.6851/-0.3801`, pitch MAE is
`1096.19/1409.92/867.13` cents, and lyric similarity is `0.0000/0.1111/0.5000`.
Its 3-second held-note medians are `148.37/136.63/139.61 Hz`, not C4.  See
`artifacts/reports/hybrid_residual_flow_evaluation.json` and
`artifacts/reports/hybrid_residual_flow_sustained.json`.  Do not use it as the
quality candidate.

## Dynamic quality-runtime gate: passed

The runnable `hybrid-soulx-phrase` backend was exercised through the public
CLI, not through a pre-rendered fixture.  It generates one complete lyric
phrase with ACE-Step, then uses SoulX-Singer with the full explicit 50 Hz
score contour.  `scripts/evaluate_soulx_score_probe.py --runtime-smoke`
measured those CLI outputs.  The predeclared gate is correlation >= 0.90,
note MAE <= 100 cents, held-note CV <= 0.10, and lyric similarity >= 0.50.

| Language | F0 corr | pitch MAE | held-note F0 CV | WavLM | ASR similarity | gate |
|---|---:|---:|---:|---:|---:|---|
| KO | 0.9942 | 11.08 cents | 0.0049 | 0.6670 | 1.0000 | pass |
| EN | 0.9637 | 21.52 cents | 0.0056 | 0.6180 | 0.7105 | pass |
| JA | 0.9917 | 12.63 cents | 0.0049 | 0.5698 | 0.6000 | pass |

Evidence: `artifacts/reports/soulx_runtime_smoke.json`.  A separate thin
package, `artifacts/package/gyu-hybrid-singer-v0.3-quality-runtime.zip`, was
unzipped and ran its `run.sh` successfully, producing a 48 kHz mono WAV.  It
deliberately excludes the 13 GB Apache-2.0 upstream ACE-Step/SoulX weights;
the README requires the compatible external cache and SoulX Python runtime.
This is quality-runtime evidence, **not** a claim that the compact v0.3
checkpoint passed quality or that this is a self-contained production package.
