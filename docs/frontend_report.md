# Frontend report

`gyu_singer.frontend.phonemize` supports `ko`, `en`, `ja`, emitting stable phoneme IDs, language IDs, syllable/word boundaries, inferred-label flags, and ten language-aware features. Korean decomposes Hangul onset/nucleus/coda and marks tense/aspirated onset classes. Japanese expands supported Kanji phrases to readings, then decomposes kana into consonant-vowel phonemes with mora, long-vowel, geminate and moraic-nasal flags; unknown Kanji is explicitly `inferred`. English uses a frozen CMU pronunciation subset for all manifest words, emitting ARPAbet-like phones and lexical stress values; unknown spelling fallback is explicitly marked `inferred`. `build_phrase_frames` exposes per-note phoneme durations, frame-level phoneme-to-note mapping, note sequence, and hard/slur boundary type rather than stretching whole phrase text across every note.
